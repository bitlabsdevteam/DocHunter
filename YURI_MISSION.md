# Yuri Mission

## Product Mission
Design and build an AI Agentic Healthcare Locator for Japan based on the provided PRD.

## Design Perspective
Think like an AI CTO at OpenAI:
- architect for reliability, safety, modularity, and extensibility
- choose practical tool boundaries
- separate UX, orchestration, retrieval, booking, and compliance concerns cleanly
- avoid fake certainty in healthcare settings
- optimize for strong agentic flow without overcomplicating the first version

## Research Requirements
- Use Perplexity for current best-practice research
- Use GitHub for architecture/code workflow
- Prioritize realistic Japanese healthcare discovery and booking constraints
- Only check in and push code related to the AI Agentic Healthcare Locator solution
- Never check in or push unrelated OpenClaw source/workspace files
- Target GitHub repo for Yuri's project code: https://github.com/bitlabsdevteam/DocHunter

## Initial Deliverables
1. System architecture
2. Agent/tool design
3. Data model proposal
4. MVP implementation plan
5. Key risks / open questions
6. Progressive roadmap to production
